tailwind CSS : CSS 프레임워크(유틸리티 퍼스트) - 기존 CSS방식으로 스타일을 적용할때에는 코드가 매우 길어지는점이 있음. - tailwind를 사용하게 될경우는 한줄로 매우 긴 CSS를 정리하는것또한 가능하다는점이 있음. - 처음 코드를 볼때에는 거부감이 들수 있음.(지저분해보임) tailwind의 장점 : 클래스명을 만들때 편함, CSS 경량화, 안정성 - 클래스명 : 현업에서는 10명이보면 8.5명은 이해가 바로 가능한 네이밍이 필요 (tailwind를 쓰면 이런 고민을 할 이유가 적어짐.) - css 경량화 : 전통적인 css를 사용할경우 새 기능 추가마다 css파일이 점점 커지게됨. -> tailwind같은 유틸리티성 개발을 진행하면 재사용성을 증가시킬수 있음. 재사용성 : 코드 한번 만들어 두고 사골국 먹듯이 우려먹는거. - 안정성 : css는 기본적으로는 전역 -> 파일이 많아지면 어떤 요소에 어떻게 영향을 줄지 알수 없음. -> tailwind를 사용하게되면 다른 파일에 영향을 줄 걱정은 x bootstrap과의 차이점 : - bootstrap은 속도에 초점을 맞춘 프레임워크 (자바스크립트 고려를 많이함/ 플러그인 제공) - tailwind는 유연성에 초점을 맞춘 프레임워크 결론적으로 프레임워크들은 생산성 증대에 초점을 맞춰뒀음. tailwind는 인라인 스타일 시트로 착각할수 있지만 핵심은 유틸리티 클래스 타입임. - 반응형 디자인, hover, focus, darkmode, 스타일 재사용, 함수, 사용자 정의 tailwind 속성정리(클래스명으로 표기.) 1. display - block(display:block), inline, flex, grid, hidden(display:none) 2. 포지셔닝 - static, relative, absolute, fixed, sticky(position:sticky) top-0, right-4(1rem), bottom1/2(bottom:50%), left-full(100%) 3. flex - flex-row(flex-direction : row) - flex-col(flex-direction : column) - justify-center(justify-content : center) - items-center(align-items : center) - flex-grow-(flex-grow:1) 번외 : grid-cols-3(3개의 열) grid-rows-4(4개의 행) gap-4(간격) grid-cols-aut-fit() 4. 너비/높이(w : width, h:height) - w-full(100% 너비) - h-screen(화면 높이) - max-w-md(최대 너비) - min-h-screen(최소 화면 높이) - w-1/2(50%) - h-3/4(75%) 5. 여백(박스모델) - p-4(패딩 전체 1rem / 16px) - px-6(수평 패딩 1.5rem / 24px) 6. 텍스트 text-red-500 : color:red - 폰트 크기와 스타일 text-xs : font-size : 0.75rem text-sm : font-size : 0.875rem text-base : font-size: 1rem text-lg : font-size: 1.125rem font-bold : font-weight:700 underline : text-decoration : underline text-center : text-align : center 7. 배경 bg-gray bg-opracity-50 : background-opacity: 0.5; 8. 테두리 border rounded : border-radius : 0.25rem rounded-full = border-radius : 9999px; 9. 그림자 .shadow : box-shadow
